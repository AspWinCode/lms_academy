# Спринты: иерархический курс (Stepik-подобная модель)

## Спринт 1 — Базовая курсовая модель (Backend)

**Цель:** курс → модуль → тема → шаги (теория + задача); API для админа и студента; прогресс по курсу/теме.

### Backend
- [x] Расширить модель Course: slug, short_description, sort_order, is_visible, archived_at
- [ ] Модель CourseNode (дерево: module, submodule, topic, subtopic)
- [ ] Модель TopicStep (theory, task с task_id)
- [ ] Модели UserCourseProgress, UserNodeProgress
- [ ] Миграция 003
- [ ] Admin API: CRUD курсов, дерево, узлы, шаги, создание задачи в теме
- [ ] Student API: список курсов, курс с деревом, тема, шаги, complete шага, прогресс
- [ ] Сохранить текущую логику submissions/judger без изменений

### Критерии приёмки
- Админ может создать курс, модули, темы, шаги (теория/задача), задачу в теме
- Студент видит список опубликованных курсов, открывает курс, тему, шаги; отмечает теорию просмотренной; решает задачу; видит прогресс

## Спринт 2 — Админка: редактор курса (Frontend)

**Цель:** страница курсов админа, редактор курса с деревом слева и формой справа.

### Frontend Admin
- [ ] Страница списка курсов (таблица: название, статус, модули, задачи, действия)
- [ ] Страница редактора курса: дерево структуры слева, редактор выбранной сущности справа
- [ ] Действия по узлу: добавить дочерний, редактировать, архивировать, порядок вверх/вниз
- [ ] Создание/редактирование темы, шага (теория), создание задачи из темы
- [ ] Маршруты: /admin/courses, /admin/courses/:courseId, /admin/courses/:courseId/nodes/:nodeId

### Критерии приёмки
- Админ открывает курс и видит дерево; выбирает узел и редактирует; добавляет модуль/тему/шаг/задачу; меняет порядок

---

## Спринт 3 — Студент: курсы и прохождение (Frontend)

**Цель:** студент видит курсы, открывает курс и тему в стиле Stepik.

### Frontend Student
- [ ] Страница /courses — список курсов (карточки, прогресс, «Продолжить»/«Открыть»)
- [ ] Страница курса: дерево слева, контент справа
- [ ] Страница темы: теория + список шагов; шаг-теория (текст + «Продолжить»); шаг-задача (встроенный редактор, отправка, результаты)
- [ ] Навигация «следующий/предыдущий шаг», прогресс по теме и курсу
- [ ] Маршруты: /courses, /courses/:courseId, /courses/:courseId/nodes/:nodeId, /courses/:courseId/steps/:stepId
- [ ] Вернуть пункт «Курсы» в сайдбар студента (вместо/вместе с «Задачи»)

### Критерии приёмки
- Студент заходит в «Курсы», видит список; открывает курс и тему; читает теорию, решает задачи в теме; видит прогресс

## Спринт 4 — Подмодули, подтемы, порядок и архивация

**Цель:** полная иерархия, reorder, archive/unarchive, публикация.

### Backend
- [ ] Поддержка подмодулей и подтем в дереве (уже в CourseNode)
- [ ] API reorder для узлов и шагов
- [ ] API archive/unarchive для курса, узла, шага, задачи
- [ ] Правила: при наличии прогресса/решений — архивация вместо удаления

### Frontend
- [ ] Drag-and-drop или кнопки вверх/вниз в дереве админки
- [ ] Архивация/разархивация в админке; подсветка архивных сущностей
- [ ] Студент не видит архивные и неопубликованные элементы

### Критерии приёмки
- Админ меняет порядок узлов/шагов; архивирует сущности; студент видит только опубликованное

## Спринт 5 — Расширение шагов и полировка

### Backend
- [ ] TopicStep: типы text, video, quiz (поля/логика по необходимости)
- [ ] Опционально: блокировка «следующая тема после завершения предыдущей»
- [ ] Опционально: дедлайны, домашние задания

### Frontend
- [ ] Отображение шагов text/video/quiz в теме
- [ ] Блокировка недоступных тем до завершения предыдущих (опционально)

### Критерии приёмки
- Тема может содержать текст/видео/тест; при необходимости — последовательное открытие тем
